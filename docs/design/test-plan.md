## Affected Pages/Routes

- `/` — main single-screen UI (home greeting + tap-to-talk + post-session panel). Verify state transitions between idle → recording → transcribing → thinking → speaking.
- `/login` — magic link email entry.
- `/auth/callback` — magic link verification, sets session cookie.
- `/api/session/start` — returns `{ session_id, target_item, memory_greeting }`. Auth required.
- `/api/session/turn` — streaming Claude response. Records buddy phrase to DB. Auth required.
- `/api/session/end` — returns approximate panel. Triggers background analyzer.
- `/api/deepgram/token` — mints 30s-scoped JWT for browser direct WSS. Auth required.

## Key Interactions to Verify

- **Tap-to-talk** triggers mic permission prompt (first time only). On second tap, starts recording without re-prompting.
- **First tap** unlocks iOS Safari AudioContext (tests must run on actual iOS Safari for this path).
- **Recording → speaking** transition: user stops talking → Deepgram final transcript fires → Claude call begins → TTS streams → buddy audio plays. All within ~3s first-audio latency.
- **Mid-session** user interruption (tap again while buddy speaking): cuts TTS, starts new recording.
- **Session end** shows approximate "what Buddy noticed" panel immediately; panel auto-updates when background analyzer completes (~3-5s later).
- **Returning-user memory greeting** on home screen reflects the *previous* session's target item and confidence delta.
- **Auth boundary:** all API routes reject unauthenticated requests with 401.

## Edge Cases

- **Mic permission denied:** onboarding screen explains requirement. App does not crash or loop.
- **STT connection fails mid-turn:** "Having trouble hearing you — tap to try again" with retry (max 2 retries, then graceful session end saving partial transcript).
- **Claude API 429 / 503:** "Buddy is thinking too hard" with retry once (exponential backoff). After 2 failures, end session with what's been saved.
- **TTS fails:** fall back to on-screen Dutch text with "Tap to read aloud" button.
- **Network drops during user recording:** buffer what's captured, resume upload on reconnect or bail with retry prompt.
- **Session hits 7-turn hard cap or 5-minute timer:** whichever fires first, session ends cleanly.
- **Cold start (first session ever):** no `learner_items` rows. App picks `topic_small_talk_greetings` as target. Greeting is "Let's start with..." not "Last time you...".
- **Analyzer updates confidence ONLY when attempts > 0** (REGRESSION TEST). Sessions where target wasn't attempted must not degrade confidence.
- **Beta-prior smoothing on confidence:** 1/1 yields ~0.4 not 1.0. 5/5 yields ~0.78. 0/5 yields ~0.22.
- **Anti-repetition:** buddy does not repeat identical opening phrases across 3 consecutive sessions.
- **Magic link replay:** used link cannot be replayed to forge a session.
- **Session cookie expiry:** app redirects to `/login` without crashing.

## Critical Paths

1. **First-time user end-to-end:** email → magic link → callback → first session (cold-start greeting) → full conversation (mocked LLM/STT/TTS) → session end → approximate panel visible → analyzer completes → DB has learner_items row + sessions row + buddy_phrases rows.
2. **Returning user end-to-end:** valid cookie → home screen shows memory greeting referencing yesterday's target → tap-to-talk → 7 turns or 5 min → session end → panel reflects this session's outcomes → next-morning open shows *today's* outcomes as "last time."
3. **Confidence-gated progression:** user earns ≥0.6 on 4 A1 items → A2 items become eligible in target selection. Before that, only A1 items surface.
4. **Audit trail:** every session has `prompt_log_json` filled with per-turn system prompts + responses, enabling replay + prompt iteration.
5. **Cost guard:** unauthenticated requests to `/api/*` return 401 without touching Deepgram/OpenAI/Anthropic. No API spend triggered by crawlers or curious visitors.

## LLM Eval Suites (run manually, not in CI for V1)

- **Eval 1: Buddy register (10 prompts).** Feed 10 A1/A2 conversational situations into prompt-composer + Claude. Hand-judge: does the reply use A1/A2 vocabulary? Short sentences? Avoids bijzin word order? Corrects previous error (if any) gently in the reply without interrupting?
- **Eval 2: Analyzer accuracy (10 hand-labeled transcripts).** Pre-label 10 mock transcripts with ground-truth `{attempts, successes, errors[]}` for the target item. Run analyzer. Compare. Score precision/recall on attempt detection + error categorization.
